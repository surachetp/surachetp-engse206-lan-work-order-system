# Pre-learning — ต้องทำก่อน Day 1

## เป้าหมาย

ก่อนเริ่ม Day 1 คุณต้องมีกรณีศึกษาที่ **เล็กพอ ปลอดภัยต่อข้อมูล และมีหลักฐานตั้งต้น** เพื่อให้เวลาในชั้นเรียนใช้กับการวิเคราะห์และออกแบบ ไม่ใช่เพิ่งเริ่มคิดหัวข้อ

## งานที่ต้องทำ

| ลำดับ | ไฟล์ | เวลาประมาณ | ผลลัพธ์ |
|---|---|---:|---|
| 1 | [`P00_case_selection_proposal.md`](P00_case_selection_proposal.md) | 45–60 นาที | case proposal + scope + evidence plan |
| 2 | [`P00_initial_context_and_ethics.md`](P00_initial_context_and_ethics.md) | 30–45 นาที | stakeholder/context เบื้องต้น + confidentiality check |
| 3 | [`P00_readiness_check.md`](P00_readiness_check.md) | 20–30 นาที | self-check เรื่อง problem/requirement/design |
| 4 | [`P00_submission_checklist.md`](P00_submission_checklist.md) | 10 นาที | ตรวจความพร้อมและ commit |

## เกณฑ์ผ่านก่อน Day 1

- ระบุ “ปัญหาใน workflow” ได้ ไม่ใช่แค่ชื่อระบบที่อยากสร้าง
- มี stakeholder อย่างน้อย 2 กลุ่ม
- มี scope/non-scope เบื้องต้น
- มีแผนเก็บ evidence ที่ไม่ละเมิดความลับ
- รับรองว่าไม่ใส่ข้อมูลอ่อนไหวใน repository หรือ AI

> หากยังไม่มี case ที่พร้อม ให้ใช้ **Campus Resource Booking** เป็น fallback ตามที่ผู้สอนกำหนด
