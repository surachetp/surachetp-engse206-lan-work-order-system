# Day 1 — Problem Discovery, Stakeholder and Scope

โฟลเดอร์นี้ใช้สำหรับ **กิจกรรมในชั้นเรียน Day 1** และ **งานที่ต้องกลับไปทำต่อหลังเรียน**

## เป้าหมายของ Day 1

เมื่อจบวันเรียนนี้ นักศึกษาควรจะสามารถ

1. แยกความต่างของ **problem, requirement, specification, design และ implementation** ได้
2. วิเคราะห์ **business goal, user goal, stakeholder, scope, constraint และ assumption** ของกรณีศึกษาได้
3. ระบุประเด็น **ethics, privacy และการใช้ AI อย่างรับผิดชอบ** ได้
4. จัดทำชิ้นงานตั้งต้นของทีมเพื่อไปต่อใน Day 2 ได้

## ลำดับการทำงานในชั้นเรียน (แนะนำ)

1. อ่านโจทย์กรณีศึกษา/กรณีของทีมตนเองอย่างรวดเร็ว
2. กำหนดบทบาทในทีมชั่วคราว เช่น facilitator, note taker, presenter, reviewer
3. ทำกิจกรรม A01 → A02 → A03 ตามลำดับ
4. เติมข้อมูลลงในเอกสาร P01 และ P02 พร้อมกันไปด้วย
5. แลกงานกับอีกทีมหนึ่งโดยใช้ A04
6. กลับไปปรับเอกสารตาม feedback และส่งตามเวลาที่ผู้สอนกำหนด

## ตัวอย่างกรณีศึกษาอ้างอิง

หากยังไม่มี case ของตนเอง ให้ใช้กรณีศึกษา **Campus Resource Booking** เป็นตัวอย่าง โดยคิดในระดับ workflow ย่อย เช่น

- การจองห้องประชุม/ห้องเรียนพิเศษ
- การจองอุปกรณ์ เช่น โปรเจคเตอร์ ไมค์ หรือชุดทดลอง
- การอนุมัติคำขอจองโดยเจ้าหน้าที่
- การคืนทรัพยากรและรายงานปัญหาหลังใช้งาน

## กติกาสำคัญ

- ห้ามใส่ข้อมูลลับ ข้อมูลส่วนบุคคล หรือเอกสารภายในของโรงงาน/หน่วยงานลงใน repository
- ใช้นามแฝงแทนชื่อบุคคล หน่วยงาน หรือระบบจริงเมื่อจำเป็น
- ทุกข้อสรุปควรระบุว่าเป็น **fact / assumption / open issue** ให้ชัด
- ถ้าใช้ AI ช่วยร่าง ให้บันทึกไว้ใน `AI_USAGE_LOG.md`
