# A05 — Day 1 Take-home Work

> งานกลับไปทำหลัง Day 1 เพื่อเตรียมพร้อมสำหรับ Day 2

## เป้าหมาย

- ปรับชิ้นงาน Day 1 ให้ชัดขึ้นจาก feedback
- เตรียมฐานข้อมูลตั้งต้นสำหรับการเก็บ requirement ใน Day 2
- อ่านเนื้อหาล่วงหน้าเรื่อง elicitation และ evidence logging

## งานที่ต้องทำ

| งาน | รายละเอียด | เวลาประมาณ | หลักฐาน |
|---|---|---:|---|
| 1. ปรับ `P01_problem_brief.md` | ปรับ problem statement, goals, pain points, success indicators | 45–60 นาที | commit/update file |
| 2. ปรับ `P02_stakeholder_context_scope.md` | ปรับ stakeholder map, context, in/out scope, constraints | 45–60 นาที | commit/update file |
| 3. เตรียมข้อมูลโดเมน | ค้นข้อมูลจากแหล่งสาธารณะ/ความทรงจำจากงานจริง โดยไม่เปิดเผยข้อมูลลับ | 30–45 นาที | note ใน references/ หรือ evidence/ |
| 4. เรียน microlearning | ทบทวน elicitation, interview, evidence logging | 45–60 นาที | note สั้น ๆ / checklist |
| 5. เตรียมคำถาม Day 2 | ระบุว่าอยากรู้อะไรเพิ่มจาก stakeholder | 20–30 นาที | bullet list 5–8 ข้อ |

## Checklist ก่อนขึ้น Day 2

- [ ] Problem statement ชัดขึ้นแล้ว
- [ ] stakeholder ที่จะสัมภาษณ์/จำลองบทบาทถูกเลือกแล้ว
- [ ] รู้ว่าอะไรคือ assumption และ open issue ของทีม
- [ ] มีคำถามตั้งต้นสำหรับ Day 2 อย่างน้อย 5 ข้อ
- [ ] commit งาน Day 1 เรียบร้อยแล้ว

## ชื่อ commit ที่แนะนำ

```bash
git add .
git commit -m "D1: frame problem, stakeholder and scope"
```

## สิ่งที่ควรพกมาใน Day 2

- problem brief และ stakeholder map ฉบับล่าสุด
- รายชื่อคำถามที่อยากถาม stakeholder
- แผนว่าใครจะรับบท interviewer / note taker / observer
