# P01 — Problem Brief

## Problem statement

ปัจจุบันพนักงานแจ้งปัญหาด้านคอมพิวเตอร์ ระบบเครือข่าย และอุปกรณ์ IT ผ่านหลายช่องทาง เช่น โทรศัพท์ อีเมล และแชต ทำให้ข้อมูลไม่เป็นมาตรฐาน ติดตามงานได้ยาก และผู้แจ้งไม่ทราบสถานะการดำเนินงาน ส่งผลให้การแก้ไขปัญหาล่าช้าและเกิดการสื่อสารซ้ำซ้อน

## Target outcome

มีระบบกลางสำหรับรับแจ้งปัญหา ติดตามสถานะ และบริหารจัดการ Work Order ที่ช่วยให้ผู้ใช้งานและทีม IT สามารถสื่อสาร ติดตาม และตรวจสอบประวัติการดำเนินงานได้อย่างมีประสิทธิภาพ

## Business/User Goals

### Business Goals
- ลดการสูญหายของข้อมูลการแจ้งซ่อม
- เพิ่มประสิทธิภาพในการติดตามและบริหารจัดการงานของฝ่าย IT
- มีข้อมูลประวัติการซ่อมเพื่อใช้ในการวิเคราะห์และวางแผนปรับปรุงบริการ

### User Goals
- แจ้งปัญหาได้ง่ายจากช่องทางเดียว
- สามารถตรวจสอบสถานะงานได้ด้วยตนเอง
- ได้รับการแก้ไขปัญหาอย่างรวดเร็วและโปร่งใส

## Primary Users

| User | Goal | Pain point |
|---|---|---|
| User Requester (ผู้แจ้งซ่อม) | แจ้งปัญหาและติดตามสถานะงาน | ไม่ทราบสถานะงานและต้องสอบถามหลายครั้ง |
| IT Support | รับงานและจัดการงานซ่อมได้อย่างเป็นระบบ | ข้อมูลที่ได้รับไม่ครบถ้วนและติดตามงานยาก |
| IT Manager | ติดตามภาพรวมและประสิทธิภาพการทำงาน | ไม่สามารถตรวจสอบสถานะงานทั้งหมดได้ง่าย |
| Purchasing Officer | ดำเนินการจัดซื้ออุปกรณ์หรืออะไหล่ | ขาดข้อมูลอ้างอิงเกี่ยวกับความต้องการใช้งาน |
| Warehouse Staff | จัดการสต็อกอะไหล่และอุปกรณ์ | ไม่ทราบการใช้อะไหล่ที่เกี่ยวข้องกับงานซ่อม |

## Success Indicators

- ผู้ใช้สามารถตรวจสอบสถานะงานได้ด้วยตนเองโดยไม่ต้องโทรสอบถาม
- ทุกคำขอแจ้งซ่อมมีหมายเลขอ้างอิงและประวัติการดำเนินงาน
- IT Support สามารถติดตามงานค้างและงานที่ได้รับมอบหมายได้ง่าย
- ข้อมูลการแจ้งซ่อมถูกจัดเก็บในระบบกลางและค้นหาได้
- ผู้บริหารสามารถเห็นภาพรวมสถานะงานและปัญหาที่เกิดขึ้นได้

## Assumptions

| Assumption | Impact if wrong | How to verify |
|---|---|---|
| พนักงานทุกคนสามารถเข้าถึงระบบผ่านเครือข่ายองค์กรได้ | ผู้ใช้บางส่วนไม่สามารถใช้งานระบบได้ | สอบถามฝ่าย IT Infrastructure และทดสอบการเข้าถึง |
| ผู้ใช้งานมีบัญชีองค์กรสำหรับเข้าสู่ระบบ | ต้องพัฒนาระบบจัดการผู้ใช้เพิ่มเติม | ตรวจสอบข้อมูลจากฝ่าย HR และ IT |
| ทีม IT Support พร้อมเปลี่ยนมาใช้ระบบใหม่ | ระบบอาจไม่ได้รับการยอมรับหรือใช้งานจริง | สัมภาษณ์และเก็บ Feedback จากทีม IT |
| ข้อมูลอุปกรณ์และทรัพย์สิน IT มีการจัดเก็บอยู่แล้ว | ไม่สามารถเชื่อมโยงงานซ่อมกับอุปกรณ์ได้ | ตรวจสอบฐานข้อมูลทรัพย์สินขององค์กร |
| องค์กรอนุญาตให้เก็บประวัติการแจ้งซ่อมในระบบได้ | อาจมีข้อจำกัดด้านนโยบายหรือความเป็นส่วนตัว | ตรวจสอบนโยบายด้านข้อมูลขององค์กร |
