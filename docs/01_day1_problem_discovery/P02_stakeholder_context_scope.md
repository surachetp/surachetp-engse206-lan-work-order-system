# P02 — Stakeholder, Context and Scope

## Stakeholder Map

![Stakeholder Map](ENGSE206-1-69%20-%20Surachet%20P.%20(2).jpg)

| Stakeholder | Role / Interest | Goal | Concern / Conflict |
|------------|----------------|------|-------------------|
| User Requester | ผู้แจ้งปัญหาด้าน IT | แจ้งปัญหาได้ง่ายและติดตามสถานะได้ | งานล่าช้า ไม่ทราบสถานะ |
| IT Support | ผู้รับผิดชอบแก้ไขปัญหา | จัดการงานได้เป็นระบบ | ข้อมูลแจ้งซ่อมไม่ครบถ้วน |
| IT Manager | ผู้ควบคุมและติดตามงาน | ตรวจสอบประสิทธิภาพทีมงาน | ไม่สามารถติดตามภาพรวมงานได้ |
| Purchasing | จัดซื้ออุปกรณ์ IT | จัดซื้ออุปกรณ์ได้ถูกต้องและรวดเร็ว | ข้อมูลความต้องการไม่ชัดเจน |
| Warehouse / Store | จัดการสต็อกอะไหล่และอุปกรณ์ | ควบคุมจำนวนอุปกรณ์คงเหลือ | สต็อกไม่เพียงพอ |
| Management | ผู้บริหาร | ตรวจสอบ KPI และคุณภาพการให้บริการ | ขาดข้อมูลสำหรับตัดสินใจ |
| System Administrator | ดูแลระบบและสิทธิ์ผู้ใช้งาน | ระบบทำงานได้อย่างต่อเนื่อง | ความปลอดภัยและความถูกต้องของข้อมูล |

## System Context

ระบบ LAN Work Order System เป็นระบบกลางสำหรับรับแจ้งปัญหา ติดตามสถานะงานซ่อม บันทึกประวัติการทำงาน และจัดทำรายงานสำหรับฝ่าย IT และผู้บริหาร

### External Actors

#### User Requester
- ส่งคำขอแจ้งปัญหา
- ตรวจสอบสถานะงาน
- รับการแจ้งเตือนผลการดำเนินงาน

#### IT Support
- รับและดำเนินการแก้ไขปัญหา
- อัปเดตสถานะงาน
- ปิดงานเมื่อดำเนินการเสร็จสิ้น

#### IT Manager
- ติดตามสถานะงานทั้งหมด
- ตรวจสอบประสิทธิภาพการทำงาน
- ดูรายงานและ KPI

#### Purchasing
- รับคำขอจัดซื้ออุปกรณ์
- ตรวจสอบงบประมาณและดำเนินการจัดซื้อ

#### Warehouse / Store
- ตรวจสอบสต็อกอุปกรณ์
- เบิกจ่ายอุปกรณ์ให้เจ้าหน้าที่

### Inputs

- ข้อมูลผู้แจ้งปัญหา
- รายละเอียดปัญหา
- รูปภาพหรือเอกสารแนบ
- ระดับความสำคัญของงาน
- ข้อมูลการอัปเดตสถานะงาน
- ข้อมูลการเบิกใช้อุปกรณ์

### Outputs

- หมายเลข Work Order
- สถานะการดำเนินงาน
- ประวัติการซ่อมและการให้บริการ
- รายงานสรุปงานประจำวัน/เดือน
- รายงานค่าใช้จ่ายและ KPI

## Scope

### In Scope

- สร้างใบแจ้งซ่อม (Work Order)
- บันทึกข้อมูลผู้แจ้งและรายละเอียดปัญหา
- แนบรูปภาพหรือเอกสารประกอบ
- มอบหมายงานให้เจ้าหน้าที่ IT
- อัปเดตสถานะงาน
- ปิดงานและจัดเก็บประวัติ
- ค้นหาข้อมูลย้อนหลัง
- สรุปรายงานงานซ่อมเบื้องต้น
- จัดการสิทธิ์ผู้ใช้งาน

### Out of Scope

- ระบบ ERP และการจัดซื้อเต็มรูปแบบ
- ระบบบัญชีและการเงิน
- ระบบ HR
- ระบบบริหารคลังสินค้าแบบละเอียด
- Mobile Application
- AI วิเคราะห์ปัญหาอัตโนมัติ
- การเชื่อมต่อระบบภายนอกขององค์กร

## Constraints and Ethics/Privacy

| Constraint / Issue | Impact | Response |
|-------------------|---------|----------|
| งบประมาณจำกัด | ไม่สามารถพัฒนาฟังก์ชันทั้งหมดได้ | พัฒนาเฉพาะฟังก์ชันสำคัญก่อน |
| ระยะเวลาโครงการจำกัด | อาจส่งมอบงานล่าช้า | กำหนดขอบเขตโครงการให้ชัดเจน |
| ผู้ใช้มีทักษะด้าน IT แตกต่างกัน | อาจใช้งานระบบได้ยาก | ออกแบบ UI ให้เรียบง่ายและใช้งานง่าย |
| ข้อมูลส่วนบุคคลของพนักงาน | เสี่ยงต่อการรั่วไหลของข้อมูล | กำหนดสิทธิ์การเข้าถึงข้อมูล |
| ความปลอดภัยของข้อมูล | ข้อมูลสูญหายหรือถูกแก้ไข | สำรองข้อมูลและบันทึกการใช้งาน |
| การใช้ AI ในอนาคต | อาจให้ผลลัพธ์ไม่ถูกต้อง | ให้มนุษย์ตรวจสอบก่อนตัดสินใจ |
| ความเป็นส่วนตัวของผู้ใช้งาน | ข้อมูลถูกเข้าถึงโดยผู้ไม่มีสิทธิ์ | ใช้ระบบ Login และ Role-based Access Control |
