# P01 — Project Alignment & Discovery (Miro Workshop Summary)

## 👤 IT Manager Persona

### 📌 Role (บทบาทหน้าที่)
* **ควบคุมและตรวจสอบงาน:** กำกับดูแลการดำเนินงานของทีม IT ให้เป็นไปตามแผน
* **อนุมัติงบประมาณ:** มีอำนาจตัดสินใจและอนุมัติค่าใช้จ่ายในแผนก

### 🎯 Goal (เป้าหมายที่ต้องการ)
* **ตรวจสอบต้นทุนได้:** เห็นตัวเลขค่าใช้จ่ายที่ชัดเจน โปร่งใส
* **ติดตามสถานะงานทั้งหมดได้:** มองเห็นภาพรวม (Overview) ของโปรเจกต์และงานทั้งหมดในมือทีม
* **ดูรายงานสรุปได้:** มีรายงานที่สรุปข้อมูลสำคัญเพื่อนำไปใช้ตัดสินใจต่อได้ทันที

### ⚠️ Concern (ความกังวล / Pain Points)
* **ค่าใช้จ่ายไม่ถูกต้อง:** กลัวข้อมูลการเงินคลาดเคลื่อน หรือมีการใช้งบเกินโดยไม่รู้ล่วงหน้า
* **ไม่มีข้อมูลย้อนหลัง:** ข้อมูลในอดีตสูญหาย หรือค้นหายาก ทำให้เปรียบเทียบแนวโน้ม (Trends) ไม่ได้
* **รายงานจัดทำยาก:** ต้องใช้เวลาและแรงงานมากในการดึงข้อมูลมาทำ Report แบบ Manual

### ❓ Day 2 Questions (คำถามสำคัญสำหรับวันถัดไป)
1. 📊 **ต้องการรายงานแบบใดบ้าง?** (e.g., รายงานค่าใช้จ่ายรายเดือน, รายงานประสิทธิภาพทีม, รายงานความคืบหน้าโปรเจกต์)
2. 🔄 **ต้องการอนุมัติงานผ่านระบบโดยตรงหรือไม่?** (e.g., ระบบ Approve งบประมาณ, ระบบตรวจรับงานผ่านแอป/อีเมล)
3. 📈 **ต้องการ Dashboard หน้าจอสรุปผลแบบไหนบ้าง?** (e.g., Dashboard สรุป Budget vs Actual, Dashboard ติดตาม Ticket/Task status)

## 🎯 Project Goals & Strategy

* **Problem Statement:** ปัจจุบันทีม IT Support ข้อมูลกระจัดกระจาย
* **Target Outcome:** มีระบบกลางจัดการงาน IT
* **Business Goals:**
  - ลดเวลาทำรายงาน
  - ควบคุมต้นทุนได้
* **User Goals:**
  - บันทึกงานง่าย
  - ค้นหางานย้อนหลังได้
* **Pain Points:**
  - ข้อมูลแยกหลายไฟล์
  - ต้องคำนวณต้นทุนเอง
  - ติดตามสถานะงานยาก
* **Success Indicators:**
  - ค้นหางานจบในระบบเดียว
  - สรุปค่าใช้จ่ายได้เร็วขึ้น
* **Assumptions:**
  - ทุกคนใช้งานผ่าน Browser
  - มีราคามาตรฐานรองรับ
* **Open Issues:**
  - ต้องมีระบบอนุมัติงานไหม?
  - ต้องมี Email แจ้งเตือนไหม?

# P02 — Stakeholder, Context and Scope

## 👥 Stakeholder Map

| Stakeholder | Role / Interest | Goal | Concern / Conflict |
|------------|----------------|------|-------------------|
| User Requester | ผู้แจ้งปัญหาด้าน IT | แจ้งปัญหาได้ง่ายและติดตามสถานะได้ | งานล่าช้า ไม่ทราบสถานะ |
| IT Support | ผู้รับผิดชอบแก้ไขปัญหา | จัดการงานได้เป็นระบบ | ข้อมูลแจ้งซ่อมไม่ครบถ้วน |
| IT Manager | ผู้ควบคุมและติดตามงาน | ตรวจสอบประสิทธิภาพทีมงาน | ไม่สามารถติดตามภาพรวมงานได้ |
| Purchasing | จัดซื้ออุปกรณ์ IT | จัดซื้ออุปกรณ์ได้ถูกต้องและรวดเร็ว | ข้อมูลความต้องการไม่ชัดเจน |
| Warehouse / Store | จัดการสต็อกอะไหล่และอุปกรณ์ | ควบคุมจำนวนอุปกรณ์คงเหลือ | สต็อกไม่เพียงพอ |
| Management | ผู้บริหาร | ตรวจสอบ KPI และคุณภาพการให้บริการ | ขาดข้อมูลสำหรับตัดสินใจ |
| System Administrator | ดูแลระบบและสิทธิ์ผู้ใช้งาน | ระบบทำงานได้อย่างต่อเนื่อง | ความปลอดภัยและความถูกต้องของข้อมูล |
